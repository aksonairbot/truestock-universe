Amit prefers Asana-inspired interaction patterns on the SeekPeek dark palette. The dark chrome is the platform's default identity as "the core engine that runs the company."

**Why:** SeekPeek's visual identity is built around the dark palette with cosmic/space theme (purple gradients, aurora banners). Asana's value is the interaction model, not the visual skin.

**How to apply:**
- Use Asana interaction patterns: view tabs, sectioned/grouped lists, inline + add, toolbar with filter/sort/group/search, sliding detail pane
- Keep dark palette tokens as default (`--bg`, `--panel`, `--accent`, `--accent-2`, etc.)
- Cosmic/space theme on hero banners and landing page
- Light mode IS now built and deployed (May 2026) — both web and mobile have full light theme
- Hero banner style: aurora gradient with big typography (42px number), NO circular progress ring — approved design
- Mobile app mirrors web design language: same colors, Poppins font, card styles
- Product name is "SeekPeek" (renamed from Skynet) — use this everywhere in UI
- Landing page at /welcome with cosmic mockup, features, CTA
